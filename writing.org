Depth signal, not a blog. Below: my literate-programming projects (code that
*is* its own documentation), a selected English article, and a dated archive.

* Literate-programming projects

I work in a literate style — prose and code together — and I authored the
tooling that makes it work across several languages:

- [[https://github.com/jingtaozf/literate-lisp][literate-lisp]] (★69) — load Common Lisp code blocks straight from Org files.
- [[https://github.com/jingtaozf/literate-elisp][literate-elisp]] (★49) — the same for Emacs Lisp; the engine behind my agent docs.
- [[https://github.com/jingtaozf/literate-clojure][literate-clojure]] (★15) — literate programming for Clojure / ClojureScript.
- [[https://github.com/jingtaozf/literate-smalltalk][literate-smalltalk]] (★6) — literate programming for Smalltalk.
- [[https://github.com/jingtaozf/s-graphviz][s-graphviz]] (★26) — an s-expression representation of GraphViz DOT in Common Lisp.
- [[https://github.com/jingtaozf/emacs-agent][emacs-agent]] — a native Emacs SDK for code agents with org-mode as the interactive UI (literate Emacs Lisp); how I drive AI agents day to day.
